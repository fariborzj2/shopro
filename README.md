## 1. داشبورد

داشبورد، صفحه اصلی پنل مدیریت است که گزارشات کلیدی و خلاصه‌ای از وضعیت فروشگاه و فعالیت کاربران را نمایش می‌دهد.

**بخش‌های اصلی داشبورد:**

-   **کارت‌های KPI (شاخص کلیدی عملکرد):**
    -   **مجموع فروش امروز:** نمایش درآمد حاصل از سفارشات ثبت‌شده در روز جاری.
    -   **تعداد سفارشات امروز:** تعداد کل سفارشات موفق در روز جاری.
    -   **کاربران جدید امروز:** تعداد کاربرانی که در روز جاری ثبت‌نام کرده‌اند.
    -   **مجموع سفارشات ماه:** تعداد کل سفارشات در ماه جاری.
-   **نمودار فروش:**
    -   نمودار خطی برای نمایش روند فروش (مبلغ کل) در **۳۰ روز گذشته** به تفکیک روز.
    -   این نمودار به مدیر کمک می‌کند تا روند رشد یا افت فروش را به سرعت تحلیل کند.
-   **نمودار ثبت‌نام کاربران:**
    -   نمودار میله‌ای برای نمایش تعداد کاربران جدید در **۶ ماه گذشته** به تفکik ماه.
-   **آخرین سفارشات:**
    -   لیستی از ۱۰ سفارش آخر ثبت‌شده به همراه اطلاعات کلیدی مانند کد سفارش، نام کاربر، مبلغ و وضعیت.
-   **محصولات پرفروش:**
    -   لیستی از ۵ محصول پرفروش ماه اخیر بر اساس تعداد فروش.

**جداول دیتابیس مرتبط:** `Orders`, `Users`, `Products`

## 2. مدیریت سفارشات

این ماژول برای مشاهده، پیگیری و مدیریت سفارشات ثبت‌شده توسط کاربران طراحی شده است.

### **صفحه لیست سفارشات**

-   **توضیحات:** در این صفحه، تمام سفارشات به صورت لیست نمایش داده می‌شوند و مدیران می‌توانند آن‌ها را فیلتر و مدیریت کنند.
-   **فیلترها:**
    -   **وضعیت سفارش:** (مثلاً: در انتظار پرداخت، در حال پردازش، ارسال شده، تحویل شده، لغو شده)
    -   **بازه زمانی:** فیلتر بر اساس تاریخ ثبت سفارش.
    -   **کد سفارش:** جستجو بر اساس کد یکتای سفارش.
    -   **موبایل کاربر:** جستجو بر اساس شماره موبایل مشتری.
-   **جدول نمایش داده‌ها:**

| ستون             | توضیحات                                        | جدول دیتابیس (فیلد) |
| ----------------- | ----------------------------------------------- | ------------------- |
| **کد سفارش**     | کد یکتای سفارش برای پیگیری                     | `Orders` (order\_code)  |
| **کاربر**        | نام و شماره موبایل کاربر سفارش‌دهنده             | `Orders` (user\_id, mobile) |
| **مبلغ کل**      | مبلغ نهایی سفارش                               | `Orders` (amount)       |
| **وضعیت**        | وضعیت فعلی سفارش (با رنگ‌بندی برای خوانایی بهتر) | `Orders` (status)       |
| **زمان ثبت**     | تاریخ و ساعت ثبت سفارش                         | `Orders` (order\_time)  |
| **عملیات**       | دکمه‌هایی برای مشاهده جزئیات و تغییر وضعیت سفارش | -                   |

### **صفحه جزئیات سفارش**

-   **توضیحات:** با کلیک بر روی یک سفارش، جزئیات کامل آن نمایش داده می‌شود.
-   **بخش‌ها:**
    -   **اطلاعات کلی سفارش:** کد سفارش، وضعیت، زمان ثبت، روش پرداخت.
    -   **اطلاعات کاربر:** نام، شماره موبایل و آدرس تحویل.
    -   **لیست محصولات:** جدول محصولات سفارش داده‌شده شامل نام محصول، تعداد، قیمت واحد و قیمت کل.
    -   **اطلاعات مالی:** مبلغ کل، تخفیف استفاده‌شده و مبلغ نهایی پرداختی.

**جداول دیتابیس مرتبط:** `Orders`, `Users`, `Products`

## 3. مدیریت محصولات

این بخش شامل مدیریت محصولات و دسته‌بندی آن‌ها می‌شود.

### 3.1. لیست محصولات

-   **توضیحات:** این صفحه امکان افزودن، ویرایش و حذف محصولات را فراهم می‌کند.
-   **فیلترها:**
    -   **نام محصول:** جستجو بر اساس نام فارسی یا انگلیسی.
    -   **دسته‌بندی:** فیلتر محصولات بر اساس دسته‌بندی.
    -   **وضعیت:** (موجود، ناموجود، پیش‌نویس).
-   **جدول نمایش داده‌ها:**

| ستون          | توضیحات                                   | جدول دیتابیس (فیلد)           |
| --------------- | ------------------------------------------ | ------------------------------- |
| **تصویر**      | تصویر شاخص محصول (بند انگشتی)             | - (از جدول Products استخراج شود) |
| **نام محصول**   | نام فارسی و انگلیسی محصول                 | `Products` (name\_fa, name\_en) |
| **دسته‌بندی**   | نام دسته‌بندی اصلی محصول                    | `Products` (category\_id)       |
| **قیمت**        | قیمت فعلی و قیمت قبلی (برای نمایش تخفیف)  | `Products` (price, old\_price)  |
| **وضعیت**      | وضعیت نمایش محصول در سایت                  | `Products` (status)             |
| **موقعیت**     | ترتیب نمایش محصول در لیست‌ها               | `Products` (position)           |
| **عملیات**     | دکمه‌های ویرایش و حذف                      | -                               |

### **فرم افزودن/ویرایش محصول**

-   **فیلدها:** نام فارسی، نام انگلیسی، انتخاب دسته‌بندی، قیمت، قیمت قبلی، وضعیت، موقعیت، توضیحات محصول (با ویرایشگر متن غنی)، گالری تصاویر.

### 3.2. دسته‌بندی محصولات

-   **توضیحات:** مدیریت دسته‌بندی‌ها به صورت درختی (والد و فرزند) برای سازماندهی بهتر محصولات.
-   **جدول نمایش داده‌ها:**

| ستون         | توضیحات                                       | جدول دیتابیس (فیلد)              |
| -------------- | ---------------------------------------------- | ---------------------------------- |
| **عنوان**     | عنوان فارسی و انگلیسی دسته‌بندی                 | `Categories` (name\_fa, name\_en)  |
| **والد**       | نام دسته‌بندی والد (در صورت وجود)             | `Categories` (parent\_id)          |
| **تصویر**     | تصویر دسته‌بندی                                | `Categories` (image\_url)          |
| **وضعیت**     | فعال یا غیرفعال بودن دسته‌بندی                 | `Categories` (status)              |
| **موقعیت**    | ترتیب نمایش                                    | `Categories` (position)            |
| **عملیات**    | دکمه‌های ویرایش و حذف                          | -                                  |

### **فرم افزودن/ویرایش دسته‌بندی**

-   **فیلدها:** عنوان فارسی، عنوان انگلیسی، انتخاب والد، تصویر، وضعیت، توضیحات، یادداشت.

**جداول دیتابیس مرتبط:** `Products`, `Categories`

## 4. مدیریت کاربران

-   **توضیحات:** مشاهده و مدیریت اطلاعات کاربران ثبت‌نام کرده در سایت.
-   **فیلترها:**
    -   **نام یا شماره موبایل:** جستجوی کاربر.
    -   **وضعیت:** (فعال، مسدود).
-   **جدول نمایش داده‌ها:**

| ستون              | توضیحات                                 | جدول دیتابیس (فیلد) |
| ------------------- | ---------------------------------------- | --------------------- |
| **نام**            | نام کامل کاربر                          | `Users` (name)        |
| **موبایل**         | شماره موبایل کاربر (برای ورود و اطلاع‌رسانی) | `Users` (mobile)      |
| **تعداد سفارشات** | مجموع سفارشات ثبت‌شده توسط کاربر        | `Users` (order\_count)|
| **تاریخ عضویت**   | زمان ثبت‌نام کاربر                        | `Users` (created\_at) |
| **وضعیت**          | وضعیت حساب کاربری (فعال/مسدود)        | `Users` (status)      |
| **عملیات**         | دکمه‌های ویرایش و مشاهده سفارشات کاربر   | -                     |

### **فرم ویرایش کاربر**

-   **فیلدها:** نام، موبایل، وضعیت، یادداشت کوتاه (برای ثبت نکات مدیریتی).

**جداول دیتابیس مرتبط:** `Users`, `Orders`

## 5. مدیریت وبلاگ

این ماژول برای مدیریت محتوای سایت شامل نوشته‌ها، دسته‌بندی‌ها و تگ‌ها استفاده می‌شود.

### 5.1. لیست نوشته‌ها

-   **توضیحات:** مدیریت کامل مقالات و نوشته‌های وبلاگ.
-   **فیلترها:**
    -   **عنوان:** جستجو در عنوان نوشته‌ها.
    -   **دسته‌بندی:** نمایش نوشته‌های یک دسته‌بندی خاص.
    -   **وضعیت:** (منتشر شده، پیش‌نویس، زمان‌بندی شده).
-   **جدول نمایش داده‌ها:**

| ستون         | توضیحات                           | جدول دیتابیس (فیلد)       |
| -------------- | ---------------------------------- | --------------------------- |
| **عنوان**      | عنوان نوشته                        | `Blog Posts` (title)        |
| **نویسنده**    | ادمینی که نوشته را ایجاد کرده      | `Blog Posts` (author\_id)   |
| **دسته‌بندی**  | دسته‌بندی اصلی نوشته               | `Blog Posts` (category\_id) |
| **بازدید**     | تعداد بازدیدهای نوشته              | `Blog Posts` (views\_count) |
| **وضعیت**     | وضعیت انتشار                       | `Blog Posts` (status)       |
| **تاریخ انتشار** | زمان انتشار یافته                 | `Blog Posts` (published\_at)|
| **عملیات**     | دکمه‌های ویرایش و حذف              | -                           |

### **فرم افزودن/ویرایش نوشته**

-   **فیلدها:** عنوان، **اسلاگ (Slug)** برای URL، محتوای اصلی (با ویرایشگر متن غنی)، چکیده، انتخاب دسته‌بندی، انتخاب تگ‌ها، تصویر شاخص، وضعیت انتشار.
-   **نکات سئو:** فیلدهای اختصاصی برای `meta_title` و `meta_description` جهت بهینه‌سازی برای موتورهای جستجو.

### 5.2. دسته‌بندی وبلاگ

-   **توضیحات:** مدیریت دسته‌بندی‌های وبلاگ به صورت درختی. عملکرد این بخش مشابه دسته‌بندی محصولات است.

### 5.3. تگ‌های وبلاگ

-   **توضیحات:** مدیریت تگ‌ها برای اتصال به نوشته‌ها و بهبود دسترسی کاربران.
-   **جدول نمایش داده‌ها:** نام تگ، اسلاگ.

**جداول دیتابیس مرتبط:** `Blog Posts`, `Blog Categories`, `Blog Tags`, `Blog Post Tags`, `Admins`

## 6. مدیریت ادمین‌ها

-   **توضیحات:** مدیریت حساب‌های کاربری مدیران پنل با سطوح دسترسی مختلف.
-   **جدول نمایش داده‌ها:**

| ستون         | توضیحات                         | جدول دیتابیس (فیلد) |
| -------------- | -------------------------------- | --------------------- |
| **نام**        | نام کامل ادمین                    | `Admins` (name)       |
| **نام کاربری** | نام کاربری برای ورود به پنل      | `Admins` (username)   |
| **ایمیل**      | ایمیل ادمین                      | `Admins` (email)      |
| **نقش**        | سطح دسترسی (مثلاً: مدیر کل، نویسنده) | `Admins` (role)       |
| **آخرین ورود**  | زمان آخرین فعالیت در پنل         | `Admins` (last\_login)|
| **وضعیت**      | فعال یا غیرفعال بودن حساب        | `Admins` (status)     |
| **عملیات**     | دکمه‌های ویرایش و حذف            | -                     |

### **فرم افزودن/ویرایش ادمین**

-   **فیلدها:** نام، نام کاربری، ایمیل، رمز عبور (فقط در حالت افزودن یا درخواست تغییر)، نقش، وضعیت.

**جداول دیتابیس مرتبط:** `Admins`

## 7. طراحی و تکنولوژی

-   **UI/UX:** طراحی پنل با تمرکز بر سادگی، کاربرپسند بودن و دسترسی سریع به بخش‌های مختلف انجام شده است.
-   **فریم‌ورک CSS:** از **TailwindCSS** برای استایل‌دهی استفاده شده است که امکان توسعه سریع و طراحی کاملاً سفارشی را فراهم می‌کند.
-   **واکنش‌گرایی (Responsive):** تمام صفحات پنل برای نمایش صحیح در دستگاه‌های مختلف (موبایل، تبلت و دسکتاپ) بهینه‌سازی شده‌اند.
-   **پشته نرم‌افزاری:** این پنل برای اجرا بر روی سرور وب **Apache** با **PHP** و دیتابیس **MySQL/MariaDB** طراحی شده است.

## 8. ساختار URL و سئو

برای ایجاد URLهای خوانا، کاربرپسند و بهینه برای موتورهای جستجو، از فایل `.htaccess` برای بازنویسی URLها استفاده می‌شود.

-   **حذف `index.php`:** تمامی درخواست‌ها به فایل `index.php` اصلی هدایت می‌شوند تا ساختار URL تمیز باشد.
-   **URLهای زیبا (Pretty URLs):** به جای پارامترهای Query String، از ساختار مبتنی بر مسیر استفاده می‌شود.
    -   مثال برای محصول: `https://yourdomain.com/products/edit/12`
    -   مثال برای نوشته وبلاگ: `https://yourdomain.com/blog/posts/my-awesome-post-slug`
-   **فیلدهای سئو:** در بخش‌های مرتبط مانند محصولات، دسته‌بندی‌ها و وبلاگ، فیلدهای `slug`، `meta_title` و `meta_description` برای کنترل کامل بر روی نمایش نتایج در موتورهای جستجو تعبیه شده‌اند.
