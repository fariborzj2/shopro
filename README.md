## 1. سیستم احراز هویت

برای دسترسی به پنل مدیریت، ورود با حساب کاربری ادمین الزامی است.

-   **صفحه ورود:** فرمی برای دریافت نام کاربری و رمز عبور.
-   **امنیت:** ارتباط با سرور از طریق POST انجام می‌شود و رمزهای عبور در دیتابیس به صورت هش‌شده (`password_hash`) ذخیره می‌شوند.
-   **مدیریت نشست (Session):** پس از ورود موفق، اطلاعات ادمین در نشست ذخیره شده و تمام صفحات پنل به جز صفحه ورود، نیازمند نشست فعال هستند.
-   **خروج:** با کلیک بر روی دکمه خروج، نشست کاربر از بین رفته و به صفحه ورود هدایت می‌شود.

**جداول دیتابیس مرتبط:** `Admins`

## 2. داشبورد

داشبورد، صفحه اصلی پنل مدیریت است که گزارشات کلیدی و خلاصه‌ای از وضعیت فروشگاه و فعالیت کاربران را نمایش می‌دهد.

**بخش‌های اصلی داشبورد:**

-   **کارت‌های KPI (شاخص کلیدی عملکرد):**
    -   **مجموع فروش امروز:** نمایش درآمد حاصل از سفارشات ثبت‌شده در روز جاری.
    -   **تعداد سفارشات امروز:** تعداد کل سفارشات موفق در روز جاری.
    -   **کاربران جدید امروز:** تعداد کاربرانی که در روز جاری ثبت‌نام کرده‌اند.
    -   **مجموع سفارشات ماه:** تعداد کل سفارشات در ماه جاری.
-   **نمودار فروش:**
    -   نمودار خطی برای نمایش روند فروش (مبلغ کل) در **۷ روز گذشته** به تفکیک روز.
-   **نمودار ثبت‌نام کاربران:**
    -   نمودار میله‌ای برای نمایش تعداد کاربران جدید در **ماه جاری** به تفکیک روز.
-   **آخرین سفارشات:**
    -   لیستی از ۵ سفارش آخر ثبت‌شده به همراه اطلاعات کلیدی.

**جداول دیتابیس مرتبط:** `Orders`, `Users`

## 3. مدیریت سفارشات

این ماژول برای مشاهده، پیگیری و مدیریت سفارشات ثبت‌شده توسط کاربران طراحی شده است. لیست سفارشات به صورت صفحه‌بندی شده نمایش داده می‌شود.

### **صفحه لیست سفارشات**

-   **جدول نمایش داده‌ها:**

| ستون             | توضیحات                                        | جدول دیتابیس (فیلد) |
| ----------------- | ----------------------------------------------- | ------------------- |
| **کد سفارش**     | کد یکتای سفارش برای پیگیری                     | `Orders` (order\_code)  |
| **کاربر**        | نام و شماره موبایل کاربر سفارش‌دهنده             | `Orders` (user\_id, mobile) |
| **مبلغ کل**      | مبلغ نهایی سفارش                               | `Orders` (amount)       |
| **وضعیت**        | وضعیت فعلی سفارش (با رنگ‌بندی برای خوانایی بهتر) | `Orders` (status)       |
| **زمان ثبت**     | تاریخ و ساعت ثبت سفارش                         | `Orders` (order\_time)  |
| **عملیات**       | دکمه‌هایی برای مشاهده جزئیات و تغییر وضعیت سفارش | -                   |

## 4. مدیریت محصولات

### 4.1. لیست محصولات

-   **توضیحات:** این صفحه امکان افزودن، ویرایش و حذف محصولات را فراهم می‌کند. لیست محصولات به صورت صفحه‌بندی شده نمایش داده می‌شود.
-   **جدول نمایش داده‌ها:**

| ستون          | توضیحات                                   | جدول دیتابیس (فیلد)           |
| --------------- | ------------------------------------------ | ------------------------------- |
| **نام محصول**   | نام فارسی محصول                 | `Products` (name\_fa) |
| **دسته‌بندی**   | نام دسته‌بندی اصلی محصول                    | `Products` (category\_id)       |
| **قیمت**        | قیمت فعلی | `Products` (price)  |
| **وضعیت**      | وضعیت نمایش محصول در سایت                  | `Products` (status)             |
| **عملیات**     | دکمه‌های ویرایش و حذف                      | -                               |

### 4.2. دسته‌بندی محصولات

-   **توضیحات:** مدیریت دسته‌بندی‌ها به صورت درختی (والد و فرزند) برای سازماندهی بهتر محصولات.

**جداول دیتابیس مرتبط:** `Products`, `Categories`

## 6. مدیریت وبلاگ

### 6.1. لیست نوشته‌ها

-   **توضیحات:** مدیریت کامل مقالات وبلاگ. لیست نوشته‌ها به صورت صفحه‌بندی شده است.
-   **جدول نمایش داده‌ها:**

| ستون         | توضیحات                           | جدول دیتابیس (فیلد)       |
| -------------- | ---------------------------------- | --------------------------- |
| **عنوان**      | عنوان نوشته                        | `Blog Posts` (title)        |
| **نویسنده**    | ادمینی که نوشته را ایجاد کرده      | `Blog Posts` (author\_id)   |
| **دسته‌بندی**  | دسته‌بندی اصلی نوشته               | `Blog Posts` (category\_id) |
| **وضعیت**     | وضعیت انتشار                       | `Blog Posts` (status)       |
| **تاریخ انتشار** | زمان انتشار یافته                 | `Blog Posts` (published\_at)|
| **عملیات**     | دکمه‌های ویرایش و حذف              | -                           |

### **فرم افزودن/ویرایش نوشته**
-   **فیلدها:** عنوان، اسلاگ، محتوای اصلی، چکیده، انتخاب دسته‌بندی، **انتخاب چندگانه تگ‌ها**، تصویر شاخص و وضعیت.

### 6.2. دسته‌بندی وبلاگ

-   **توضیحات:** مدیریت کامل دسته‌بندی‌های وبلاگ (افزودن، ویرایش، حذف).

### 6.3. تگ‌های وبلاگ

-   **توضیحات:** مدیریت کامل تگ‌ها (افزودن، ویرایش، حذف) برای اتصال به نوشته‌ها.

**جداول دیتابیس مرتبط:** `Blog Posts`, `Blog Categories`, `Blog Tags`, `Blog Post Tags`, `Admins`

## 7. مدیریت ادمین‌ها

-   **توضیحات:** این بخش برای مشاهده لیست مدیران پنل است.
-   **جدول نمایش داده‌ها:** نام، نام کاربری، ایمیل، نقش و وضعیت.
-   **نکته:** به دلایل امنیتی، قابلیت افزودن، ویرایش و حذف ادمین‌ها از طریق رابط کاربری پیاده‌سازی نشده است.

**جداول دیتابیس مرتبط:** `Admins`

---

## 8. طراحی و تکنولوژی

-   **UI/UX:** طراحی پنل با تمرکز بر سادگی و کاربرپسند بودن انجام شده است.
-   **فریم‌ورک CSS:** از **TailwindCSS** برای استایل‌دهی استفاده شده است.
-   **واکنش‌گرایی (Responsive):** تمام صفحات برای نمایش صحیح در دستگاه‌های مختلف بهینه‌سازی شده‌اند.
-   **پشته نرم‌افزاری:** PHP, Apache, MySQL/MariaDB.
-   **جاوا اسکریپت:** از **Alpine.js** برای تعاملات ساده در UI (مانند منوی کشویی) و از **Chart.js** برای نمایش نمودارها استفاده شده است.

## 9. ساختار URL و سئو

-   **URLهای زیبا (Pretty URLs):** با استفاده از `.htaccess`، URLها خوانا و بهینه شده‌اند.
-   **نقشه سایت (Sitemap):** یک فایل `sitemap.xml` به صورت خودکار برای نوشته‌های وبلاگ منتشر شده، تولید و به‌روز می‌شود.
-   **فایل `robots.txt`:** این فایل برای جلوگیری از ایندکس شدن پنل مدیریت توسط موتورهای جستجو پیکربندی شده و آدرس نقشه سایت در آن مشخص شده است.
